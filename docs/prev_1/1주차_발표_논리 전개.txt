- 2013 Visibility Buffer
G-buffer에 material 대신에 triangle Index를 저장하자. (triangle(primitive) ID + instance ID/draw ID + barycentric coordinate + depth)
그리고 두 번째 패스 (shading pass)에서 해당 데이터를 사용. 무게중심 좌표로 보간.
-> GPU의 HW 래스터라이저가 보간 시에는 그냥 다른 코어를 사용하나? 혹은 보간 유닛도 내장돼 있나? 조사해보기
아무튼 b-buffer bandwidth를 절감할 수 있는 효과. 또한 material이 g-buffer format에 영향받지 않음.

material이 g-buffer format에 영향받지 않는 것에 대해선,

이걸 여러 씬에서 여러 가정을 통해 실험해보면 좋을듯

단점: 1 삼각형 ID와 vertex data 재접근이 random access 성격이 강해서 캐시 효율이 나쁠 수 있음.

- TVB 1.0
실제 첫 번쨰 실용화 형태로 생각.

1. Depth / Visibility Pass
   - opaque geometry rasterization
   - depth buffer 생성
   - pixel마다 triangle/draw/material ID 저장

2. Shading Pass
   - visibility buffer read
   - triangle ID로 vertex attributes fetch
   - barycentric interpolation
   - material evaluation
   - lighting

3. Post-processing
   - TAA, bloom, tone mapping 등


TVB shading pass는 compute shader로 구현.
해결하는 문제: 1 G-buffer bandwidth, 2 material 다양성 3 shading overdraw
한계: 1 coherence부족 2 texture derivative문제(바운더리에서 -> 보통 상황에선 어케 되는지?)
3 material shader 실행 모델 (하나의 셰이더로 처리하면 분기가 많다..) 4 투명 처리


- TVB 2.0
여러 최적화들이 붙음.
1 material/triangle binning: visible pixel 을 material별로 분류해 각 material별 work queue 생성, shading
-> divergence로 인한 병목 (긴 셰이더 실행시간)을 관측할 필요가 있겠음.

2 여러 compute shader로 더 유연하게 처리.

즉 TVB 1.0의 random access를 binning과 cluster로 coherence를 높여 해결.
또 material divergence를 work list를 만들어 해결.
