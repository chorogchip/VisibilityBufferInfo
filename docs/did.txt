[07-01 진행상황]

모델에서 Scene 전체를 그리도록 확장
- 여러 인스턴스를 그릴 수 있도록 렌더러를 확장하였습니다.
- 전역 vertex / index 버퍼에 geometry들을 담고, offset 으로 사용합니다.
- structured buffer 에 인스턴스별 데이터(월드 행렬)를 담아 사용합니다.
- 단 geometry/material을 공유하는 오브젝트끼리 묶어서 인스턴싱하는 기능은 구현되지 않았습니다.
- 각 버퍼는 아직 upload heap을 사용합니다. 추후 default heap을 사용하는 구조로 전환할 예정입니다.

- 장면은 파라미터에 따라 자동으로 생성됩니다.
- 현재 매개 변수는 다음과 같습니다.

struct SceneInfo {
	uint32_t seed;
	uint32_t sphere_count;
	uint32_t material_count;
	uint32_t geometry_count;
	float z_min;
	float z_max;
	float xy_min;
	float xy_max;
	float radius_min;
	float radius_max;
	uint32_t geometry_division_min;
	uint32_t geometry_division_max;
	uint32_t material_float4_count;  // g버퍼 장수 (미구현)
};

- 모델로 Spherified Cube를 선택한 이유는 추후 meshlet을 고려할 경우 격자 단위로 나누기 쉽다고 판단했기 때문입니다.
- 각 구의 6면은 설정한 개수의 삼각형 (geometry_division^2 * 2) 으로 분할됩니다.
- TVB에 유리한 인위적인 상황을 막기 위해 어떤 구들은 데이터는 같지만 서로 다른 주소에 별개로 존재하는 vertex/index 를 사용할 수 있습니다.

[07-03 진행상황]

렌더러 개발
- 커맨드라인 인자를 파싱해 외부에서 파라미터를 읽어오도록 했습니다.
- 설정한 조건에 따른 프레임만큼 실행한 뒤 종료하도록 만들었습니다.
- 파이썬 스크립트로 렌더러를 실행하도록 했습니다.
- 렌더링에 필요한 버퍼들을 upload heap에 올려 default heap으로 복사해 정상적인 성능으로 사용할 수 있도록 했습니다.
- readback buffer를 이용해 GPU timestamp query를 읽어올 수 있도록 했습니다.



각 방법별 렌더러 계획
- forward: material과 geometry를 순회하며 둘을 공유하는 인스턴스끼리 묶어 draw call을 하려고 합니다.
- forward + depth pre-pass : forward와 같되 geometry를 순회하며 geometry를 공유하는 인스턴스끼리 묶어 pre-pass를 계산합니다.
- deferred : material과 geometry를 순회하며 geometry pass를 그리고, shading pass는 통합된 하나를 사용합니다.
- TVB: geometry를 순회하며 visibility buffer를 만들고, 그 뒤는 같은 material끼리 묶어 draw call을 하려고 합니다.
	추후 material에 상관없이 통합된 draw call을 사용해보는 접근을 하려고 합니다.



씬 계획 - 초반에 인조 구 씬으로 다음과 같은 시도를 하려고 합니다.
- Tradeoff 관찰:  overdraw, material, G-buffer, triangle size 등등에 따라 각 알고리즘의 성능 분석, 변수에 따른 민감도 관찰
- Crossover Point 찾기: 위와 같은 맥락으로 유리해지는 알고리즘이 바뀌는 지점 찾기
- 데이터와 맞게 성능 모델을 수립
- Roofline 분석: VRAM bandwidth, ALU, texture sampling, rasterization, L2 cache 등에 의한 roofline 분석



씬 계획 - 이후 인터넷에서 구할 수 있는 씬으로 다음과 같은 시도를 하려고 합니다.
- Scene fingerprint 추출 (각 씬의 geometry/material/texture/overdraw 특성 수치화)
- 구에서 세운 모델이 현실에서도 통하는지 가설을 세우고 검증
- HW 프로파일링을 동반
- 안 통한다면 이유를 찾기
- TVB 약점, 강점 케이스 찾기
- material binning / sorting 실험 (확장) (편집됨)
